Why Each Attack Fails

⟲ Replay Attack Prevention

Every nonce is stored with a used: bool flag. When first consumed, flag → True. Any subsequent attempt returns used=True immediately. The nonce store is scoped server-side — no device can bypass it. Even with a valid MAC, replay is rejected.

👤 Spoofing Prevention

The PSK is never transmitted. Without it, an attacker cannot compute HMAC(PSK, nonce||id||ts). Even if attacker knows device_id and intercepts the nonce, without PSK the HMAC will differ. Server uses hmac.compare_digest() — constant-time, no timing leak.

⚡ MITM Prevention

Nonce is included inside the HMAC message. If attacker modifies the nonce in transit: the device computes HMAC over the original nonce, but the server checks against the tampered nonce → HMAC mismatch. Any bit-flip in the nonce is detected.
